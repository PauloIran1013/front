Projeto Integrador

Objetivos:

Os objetivos do projeto para a criação da loja de jogos Cleiton games são:

  • Desenvolver uma Plataforma Funcional e Responsiva:

    Criar um sistema acessível em diversos dispositivos que permita aos usuários visualizar e gerenciar um inventário de jogos de maneira eficiente.

  • Atender às Necessidades dos Usuários Finais:

    Garantir que o inventário ofereça uma interface intuitiva para a adição, remoção e organização dos jogos, além de permitir a visualização de detalhes e categorias.

  • Implementar Boas Práticas de Programação e Gerenciamento de Projetos:

    Utilizar padrões de codificação adequados e aplicar metodologias ágeis, como Scrum, para gerenciar o desenvolvimento e garantir a qualidade do projeto.

  • Facilitar o Aprendizado Prático:

    Proporcionar a aplicação de metodologias ágeis e práticas de desenvolvimento em um contexto real, com a implementação de funcionalidades específicas como sistemas de categorização e busca.

  • Incluir Funcionalidades Essenciais:

    : Desenvolver funcionalidades como cadastro de jogos, categorização, filtros de busca, e opções de edição e visualização de detalhes.

Briefing

O briefing para o projeto de inventário de jogos deve incluir:

  • Descrição do Projeto:

    Criação de um sistema de inventário para gerenciar uma coleção de jogos, com funcionalidades para adicionar, editar, organizar e visualizar os jogos.

  • Objetivos Esperados:

    Desenvolver uma plataforma fácil de usar e eficiente, permitindo a gestão do inventário de jogos com uma interface limpa e funcionalidades robustas.

  • Público-Alvo:

    Jogadores que desejam organizar e gerenciar suas coleções de jogos, tanto pessoais quanto profissionais.

  • Necessidades do Cliente:

    Permitir a criação e manutenção de um inventário digital, com capacidade para categorizar, buscar e visualizar jogos de forma eficiente.

  • Principais Problemas a Serem Resolvidos:

    Simplificar a gestão e organização de uma coleção de jogos, oferecendo uma interface intuitiva e funcionalidades de busca e categorização eficazes.

  • Especificações Técnicas e de Design:

    O sistema deve ser responsivo, ter um design claro e intuitivo, e suportar funcionalidades de adição, edição, categorização e busca de jogos.

Metodologia:

Uso de Scrum para desenvolvimento ágil e boas práticas de codificação para garantir qualidade e escalabilidade.

Requisitos:

  • Funcionais: Cadastro, login, busca de jogos, compra e avaliações.
  • Não Funcionais: Desempenho rápido, segurança (com criptografia e pagamentos seguros), e compatibilidade com diferentes dispositivos.

Tecnologias:

  • Frontend: HTML, CSS, JavaScript (React).

  • Backend: Node.js, Express.js.

  • Banco de Dados: PostgreSQL/MySQL.

  • Segurança: SSL/TLS, OWASP.

  • Infraestrutura: Docker, Kubernetes, AWS/GCP.

Objetivo Final:

Criar uma plataforma fácil de usar, responsiva e segura, que permita aos usuários organizar suas coleções de jogos e facilitar a compra de novos títulos.
